# Cas d'usage du système de ticketing

## Use cases principaux

1. Créer un ticket : Un utilisateur crée un nouveau ticket avec titre, description
2. Assigner un ticket : Un agent assigne un ticket à lui-même ou un autre agent
3. Démarrer le traitement : Un agent démarre le travail sur un ticket assigné
4. Modifier le statut d'un ticket : Un agent change le statut d'un ticket en démarrer, résoudre, clôturer
5. Ajouter une note : Un agent ou un utilisateur ajoute une note, un commentaire à un ticket
6. Consulter l'historique : Un utilisateur ou un agent consulte l'historique des actions d'un ticket
7. Rechercher des tickets : Un utilisateur effectue une recherche de tickets selon différents critères comme le statut, la priorité, l'agent assigné
8. Fermer un ticket : Un agent ferme un ticket une fois que le problème est résolu.
